## Project Overview

A task tracker for a single account manager handling ~40 accounts. Reads Google Sheets ("참여기업 현황판" and "[S/A] 채용 수요") in read-only mode to import account data, manages hierarchical tasks (parent tasks × per-account subtasks) with ETA tracking, and delivers Slack DM alerts for approaching or overdue deadlines. Built with a multi-user schema so it can expand to the full team later.

## Working Style

- **Conversation language**: Korean. All explanations, plan steps, and prose to the user happen in Korean.
- **File / code / comment language**: English. Function names, file names, commit messages, inline comments all in English.
- **User is non-developer** — use single-file scripts and simple HTML/CSS/JS. Avoid Docker, microservices, framework boilerplate, multi-module layouts.
- **Verify credentials before writing integration code**. Confirm the user has each API credential with a smoke test before scaffolding any integration (Sheets → Slack → Gmail → Notion → Claude API, in that order).
- **When in doubt about an ambiguous requirement, ask one clarifying question** rather than guess silently.
- **After significant changes, summarize what changed in Korean** in 1–2 sentences.
- **Google Sheets source files are read-only** — never write to or modify them under any circumstance.
- **No silent auto-save** — task candidates from Slack/Gmail/Notion must never be saved to the DB without explicit user confirmation in the review UI.
